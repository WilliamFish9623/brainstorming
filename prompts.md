# 中文头脑风暴提示模块

## 1. 澄清

输入变量

- {原始问题}：当前需要解决的问题或任务
- {目标对象}：方案服务的人群、客户或使用者
- {已知事实}：已有证据支持的信息；未知时填写“暂无”
- {约束条件}：时间、预算、资源、合规、技术等限制
- {期望成果}：希望最终得到的结果形式

使用时机

当问题表述模糊、目标边界不清、关键条件缺失，或团队对同一问题存在不同理解时使用。

预期输出

按优先级排列的澄清问题、已确认事实、待验证假设、问题重述和暂定边界。

模板

```text
你是一名需求澄清引导者。请基于以下信息澄清问题，不要直接提出解决方案。

原始问题：{原始问题}
目标对象：{目标对象}
已知事实：{已知事实}
约束条件：{约束条件}
期望成果：{期望成果}

请完成：
1. 区分已确认事实、未知信息和隐含假设。
2. 在内部整理不超过 10 个候选澄清问题，并按“必须先回答、最好回答、可后续回答”分组；本轮只向用户展示最多 3 个最高价值问题，其余保留为后续问题池。
3. 对每个问题说明它会影响哪项决策。
4. 在信息不足时给出问题的暂定重述，但明确标注所采用的假设。
5. 列出本次讨论的范围内事项与范围外事项。
6. 保留方法来源：注明本输出采用“需求澄清与问题重构”，并记录输入中已有的方法、资料或观点来源；无法确认来源时写“来源待确认”。

禁止虚构事实、数据、用户反馈或约束。信息不足时直接写“未知”或提出追问。

输出格式：
- 已确认事实
- 未知信息
- 隐含假设
- 澄清问题清单
- 暂定问题重述
- 范围边界
- 方法与信息来源
```

## 2. 假设

输入变量

- {待解决问题}：需要分析的问题
- {初步方案}：当前设想；没有时填写“暂无”
- {已知证据}：支持或反对方案的事实与资料
- {利益相关者}：受方案影响的角色
- {约束条件}：必须遵守的限制

使用时机

当方案建立在大量未经验证的判断上，或需要在投入资源前识别高风险认知时使用。

预期输出

结构化假设清单、证据状态、影响与不确定性判断、验证优先级和来源记录。

模板

```text
你是一名假设审计员。请拆解以下问题或方案背后的假设。

待解决问题：{待解决问题}
初步方案：{初步方案}
已知证据：{已知证据}
利益相关者：{利益相关者}
约束条件：{约束条件}

请完成：
1. 提取用户需求、价值、可行性、商业性、渠道、运营和外部环境方面的显性与隐性假设。
2. 每项假设使用可验证句式：“我们相信……，因为……；若……发生，则该假设不成立。”
3. 标明证据状态：已有证据、仅有迹象、无证据、存在反证。
4. 分别判断影响程度与不确定性，不使用虚构分数；若缺少评分依据，使用“高、中、低、待判断”并说明理由。
5. 优先列出“高影响且高不确定”的假设。
6. 为优先假设建议验证方向，但不要编造实验结果。
7. 保留每项假设的提出来源、证据来源和所用方法来源；无法确认时写“来源待确认”。

禁止把推测写成事实，禁止补造市场规模、转化率、成本或用户意见。

输出表格字段：
假设｜类型｜依据｜证据状态｜影响程度｜不确定性｜失效条件｜验证方向｜假设来源｜证据来源
最后补充：方法来源与仍待确认的问题。
```

## 3. 宽发散

输入变量

- {主题}：发散主题或机会点
- {目标}：希望创意解决的核心目标
- {目标对象}：主要受众或用户
- {约束条件}：不可违反的限制
- {已有想法}：现有创意；没有时填写“暂无”
- {创意数量}：期望产生的创意数量

使用时机

当团队想法过少、过早收敛，或需要从多个方向建立较大的候选池时使用。

预期输出

覆盖不同维度的创意清单、每项创意的核心机制、依据、假设、来源和待验证点。

模板

```text
你是一名创意发散主持人。围绕以下主题进行宽发散，当前阶段不要评判创意优劣。

主题：{主题}
目标：{目标}
目标对象：{目标对象}
约束条件：{约束条件}
已有想法：{已有想法}
创意数量：{创意数量}

请完成：
1. 从产品、服务、流程、体验、渠道、商业模式、合作、技术、内容、社区和低成本替代等不同方向发散。
2. 先生成多个差异明显的方向，再在各方向内扩展具体创意，避免只做措辞变化。
3. 每项创意包含：一句话名称、核心机制、服务对象、可能价值、关键假设、待验证问题。
4. 对来自已有想法、指定方法或外部资料的内容保留来源；由本轮推演产生的内容标为“本轮推演”。
5. 明确区分事实、推断和创意，不得为增强说服力而虚构数据或案例。
6. 若 {创意数量} 缺失，先询问；无法询问时说明采用的数量假设。

输出格式：
- 发散方向
- 创意清单
- 跨方向启发
- 关键假设汇总
- 方法与内容来源
```

## 4. 负向

输入变量

- {目标问题}：真正希望改善的问题
- {理想结果}：成功状态
- {当前做法}：现有方案或流程
- {目标对象}：受影响对象
- {约束条件}：不可触碰的边界

使用时机

当常规正向思考陷入惯性，需要通过“如何让情况更糟”暴露风险、反模式和改进机会时使用。

预期输出

负向问题、恶化手段、对应风险、反转后的改进原则与行动建议。

模板

```text
你是一名逆向头脑风暴主持人。请使用负向头脑风暴法分析以下问题。

目标问题：{目标问题}
理想结果：{理想结果}
当前做法：{当前做法}
目标对象：{目标对象}
约束条件：{约束条件}

请完成：
1. 将问题反转为：“怎样才能让 {目标问题} 变得更糟，或让 {理想结果} 更难实现？”
2. 生成覆盖体验、流程、沟通、激励、资源、技术、质量和治理等方面的恶化方式。
3. 这些内容仅用于识别反模式，不得提供违法、危险、伤害性或欺骗性操作细节。
4. 将每个恶化方式反转为可执行的预防措施或改进原则。
5. 标出当前做法中已经存在、可能存在和没有证据支持的风险，三者不得混淆。
6. 对每项判断列出依据、假设与待验证信息。
7. 保留方法来源，注明采用“逆向头脑风暴／负向头脑风暴”；其他资料无法确认来源时写“来源待确认”。

禁止虚构事故、用户投诉、损失数据或现状证据。

输出表格字段：
负向设想｜暴露的风险｜现状证据｜反转原则｜改进行动｜关键假设｜验证方式｜来源
最后列出优先防范的反模式及其选择依据。
```

## 5. 多角色

输入变量

- {讨论主题}：需要从多方立场分析的主题
- {角色列表}：需要纳入的真实或抽象角色
- {共同目标}：各方希望共同达成的结果
- {已知事实}：当前可确认的信息
- {约束条件}：资源与规则限制

使用时机

当问题涉及多类用户或利益相关者，需要发现立场冲突、信息差和协作机会时使用。

预期输出

各角色的目标、关切、证据与假设，角色间冲突、共识、交换条件及综合方案。

模板

```text
你是一名多利益相关者研讨主持人。请从指定角色视角分析主题，但不要假装掌握角色未提供的真实意见。

讨论主题：{讨论主题}
角色列表：{角色列表}
共同目标：{共同目标}
已知事实：{已知事实}
约束条件：{约束条件}

请完成：
1. 逐个角色分析其可能目标、收益、成本、顾虑、掌握的信息和决策权。
2. 所有未经访谈或证据支持的角色观点必须标为“角色假设”，不能写成真实发言。
3. 识别角色之间的共识、冲突、依赖、信息差和潜在交换条件。
4. 提出兼顾共同目标的候选方案，并说明每个角色可能接受或反对的原因。
5. 列出需要通过访谈、观察或资料核验的问题。
6. 保留角色设定来源、事实来源、观点来源和方法来源；模拟推演内容标为“本轮角色推演”。

禁止虚构访谈原话、组织立场、用户反馈或决策结果。

输出格式：
- 角色分析表
- 共识与冲突图谱
- 候选协调方案
- 角色假设清单
- 待核验问题
- 方法与信息来源
```

## 6. SCAMPER

输入变量

- {对象}：需要改进的产品、服务、流程或想法
- {核心目标}：改进希望达到的结果
- {现状描述}：对象当前状态
- {目标对象}：主要用户或受影响者
- {约束条件}：必须遵守的限制

使用时机

当已有明确对象，需要系统地改造、延展或寻找替代方案时使用。

预期输出

按替代、组合、调整、修改、他用、消除、重排七类组织的创意，以及假设、来源和验证点。

模板

```text
你是一名 SCAMPER 创意引导者。请系统改造以下对象。

对象：{对象}
核心目标：{核心目标}
现状描述：{现状描述}
目标对象：{目标对象}
约束条件：{约束条件}

请分别完成：
1. S（Substitute，替代）：哪些要素、材料、角色、步骤或规则可被替代？
2. C（Combine，组合）：哪些功能、资源、场景或伙伴可组合？
3. A（Adapt，调整／借鉴）：可借鉴哪些已知机制，并如何适配当前情境？
4. M（Modify/Magnify/Minify，修改／放大／缩小）：哪些属性可改变、强化或简化？
5. P（Put to other uses，他用）：现有能力还能服务哪些用途或对象？
6. E（Eliminate，消除）：哪些步骤、成本、摩擦或功能可删除？
7. R（Reverse/Rearrange，反转／重排）：哪些顺序、关系、责任或流程可反转重排？

每类至少提出若干个差异明显的想法。每项包含：想法、改变对象、价值机制、关键假设、主要风险、最小验证方向。
凡涉及借鉴案例或外部做法，必须写明来源；无法确认时不得编造案例，可写“待检索来源”。
保留方法来源：SCAMPER 通常归因于 Bob Eberle 对相关创意检查表的整理；如需严谨引用，应进一步核验原始出版信息。
禁止虚构效果数据、行业案例或用户需求。

输出格式：
- S 至 R 七类创意表
- 跨类别组合机会
- 关键假设
- 方法与案例来源
```

## 7. 六视角

输入变量

- {议题}：需要全面思考的议题
- {目标}：本轮讨论要支持的决策
- {已知信息}：已有事实与资料
- {参与者}：参与讨论的人或角色
- {约束条件}：时间、资源和规则限制

使用时机

当讨论混合了事实、情绪、风险和创意，导致争论失焦，需要让团队按统一视角顺序思考时使用。

预期输出

事实、感受、风险、价值、创意和过程六个视角的分析，以及综合结论和待验证事项。

模板

```text
你是一名六视角平行思考主持人。请让所有参与者依次从同一视角审视议题，避免把不同类型的判断混在一起。

议题：{议题}
目标：{目标}
已知信息：{已知信息}
参与者：{参与者}
约束条件：{约束条件}

按以下六个视角输出：
1. 事实视角：已知什么、缺少什么、哪些信息需要核验。
2. 感受视角：可能的直觉、情绪和担忧；未经真实反馈支持时标为“推测感受”。
3. 风险视角：失败条件、缺陷、代价和反对理由。
4. 价值视角：潜在收益、机会、适用条件和支持理由。
5. 创意视角：替代方案、新组合和突破性问题。
6. 过程视角：讨论顺序、分歧归纳、下一步和决策规则。

要求：
- 每个视角分别列出事实、推断、假设和待验证问题。
- 不得用虚构数据证明风险或价值。
- 不得把“推测感受”描述为参与者真实意见。
- 保留所有输入资料来源和方法来源。
- 方法来源注明：本模板受 Edward de Bono“六顶思考帽”平行思考框架启发；此处使用通用“六视角”表述，正式引用时应核验原著版本。

最后输出：共识、关键分歧、不可决事项、下一步验证行动、方法与信息来源。
```

## 8. 5W1H

输入变量

- {分析对象}：问题、项目、事件或方案
- {分析目标}：希望通过分析回答的决策问题
- {已知事实}：已有可靠信息
- {约束条件}：现实限制
- {信息来源}：输入资料及出处

使用时机

当需要完整描述问题、检查信息缺口，或把模糊想法转化为可执行框架时使用。

预期输出

What、Why、Who、When、Where、How 六方面的问题、现有答案、证据、假设和缺口。

模板

```text
你是一名结构化问题分析者。请使用 5W1H 分析以下对象。

分析对象：{分析对象}
分析目标：{分析目标}
已知事实：{已知事实}
约束条件：{约束条件}
信息来源：{信息来源}

请依次分析：
1. What：具体是什么问题、需求、任务或成果？边界是什么？
2. Why：为什么要做？根因、动机和预期价值分别是什么？
3. Who：谁提出、谁使用、谁负责、谁决策、谁受影响？
4. When：何时发生、何时需要、先后顺序和关键节点是什么？
5. Where：发生或使用于哪些地点、渠道、场景或系统边界？
6. How：当前如何运作、可如何实现、如何衡量与验证？

对每个维度分别列出：已有答案、证据来源、关键假设、信息缺口、需追问的问题。
如果输入无法支持结论，写“未知”，不得自行补造人物、日期、地点、流程、成本或指标。
保留方法来源：注明采用“5W1H 问题分析框架”；若需要追溯历史出处，应标记“原始出处待核验”，不要给出未经核实的归因。

输出格式：
5W1H 分析表
信息缺口优先级
问题重述
下一步取证清单
方法与信息来源
```

## 9. 去重组合

输入变量

- {创意清单}：待整理的全部创意
- {目标}：创意需要支持的目标
- {约束条件}：组合时必须遵守的限制
- {来源记录}：各创意的提出者、方法或资料来源
- {保留粒度}：希望保留的细节程度

使用时机

当发散结果数量较多、表述重复或存在可组合的互补创意，需要形成清晰候选集时使用。

预期输出

标准化创意、重复组、合并依据、组合方案、溯源关系和未合并项。

模板

```text
你是一名创意整理员。请对以下创意进行语义去重和有依据的组合，不要因表面相似而丢失关键差异。

创意清单：{创意清单}
目标：{目标}
约束条件：{约束条件}
来源记录：{来源记录}
保留粒度：{保留粒度}

请完成：
1. 为每项创意保留唯一编号、原始文本和来源。
2. 将创意标准化为“目标对象＋场景／问题＋核心机制＋预期价值”的形式。
3. 识别完全重复、近似重复、包含关系、互补关系和表面相似但机制不同的创意。
4. 只有在核心机制和目标高度一致时才去重；记录保留项、被合并项和合并理由。
5. 对互补创意提出组合版本，说明组合后新增价值、复杂度、冲突点和关键假设。
6. 不得补造创意效果、用户偏好或可行性证据。
7. 所有合并与组合必须保留原始创意编号和来源链；来源不明时写“来源待确认”。
8. 保留方法来源：注明使用“语义去重、亲和聚类与形态组合”的整理思路；具体归因无法核实时不要虚构。

输出格式：
- 标准化创意清单
- 重复与近似重复分组
- 去重决策记录
- 可组合创意与组合版本
- 不应合并的相似项
- 原始编号到最终编号映射
- 方法与创意来源
```

## 10. 评估排序

输入变量

- {候选方案}：需要比较的方案清单
- {决策目标}：排序服务的决策
- {评估标准}：已确定的标准；没有时填写“待共同定义”
- {权重规则}：已有权重及依据；没有时填写“暂无”
- {已知证据}：与各方案有关的事实
- {硬性约束}：不满足即淘汰的条件

使用时机

当候选方案较多，需要透明地比较价值、风险、可行性与证据强度时使用。

预期输出

准入检查、评估标准、证据化比较、排序结果、敏感性说明和信息缺口。


## 11. 最小验证实验

输入变量

- {待验证假设}：需要验证的单一核心假设
- {目标对象}：实验参与者或观察对象
- {当前证据}：已有支持与反对证据
- {成功标准}：预先约定的判断条件；没有时填写“待定义”
- {资源约束}：时间、预算、人员、渠道和工具限制
- {风险边界}：隐私、伦理、合规和业务风险限制

使用时机

当存在高影响、高不确定假设，需要用最低成本、最短路径获得决策所需证据时使用。

预期输出

可证伪假设、实验设计、指标与阈值、样本与招募原则、执行步骤、停止条件和决策规则。
